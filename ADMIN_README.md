# Admin README

## Роль администратора в проекте

### Обязанности администратора
Администратор отвечает за управление контентом платформы, мониторинг пользователей и обеспечение безопасности системы.

### Ключевые функции администрирования

#### 1. Управление учебным контентом
- Создание и редактирование учебных блоков (Units)
- Управление уроками (Lessons) и упражнениями (Exercises)
- Публикация/скрытие контента
- Организация контента по уровням сложности

#### 2. Мониторинг пользователей
- Просмотр всех родителей и детей на платформе
- Отслеживание прогресса детей
- Анализ статистики использования

#### 3. Система аудита
- Просмотр всех действий пользователей
- Отслеживание изменений в контенте
- Мониторинг безопасности

#### 4. Статистика платформы
- Общее количество пользователей
- Количество завершенных уроков
- Метрики вовлеченности

### API для администраторов

#### Управление контентом
```
POST /api/v1/units          # Создание учебного блока
PUT /api/v1/units/{id}      # Редактирование блока
DELETE /api/v1/units/{id}   # Удаление блока

POST /api/v1/lessons        # Создание урока
PUT /api/v1/lessons/{id}    # Редактирование урока
DELETE /api/v1/lessons/{id} # Удаление урока

POST /api/v1/lessons/{id}/exercises  # Создание упражнения
PUT /api/v1/exercises/{id}           # Редактирование упражнения
DELETE /api/v1/exercises/{id}        # Удаление упражнения
```

#### Мониторинг и статистика
```
GET /api/v1/admin/stats     # Общая статистика платформы
GET /api/v1/admin/logs      # Аудит-логи всех действий
GET /api/v1/children        # Список всех детей (с пагинацией)
GET /api/v1/parents         # Список всех родителей
```

### Безопасность и доступ
- Администраторы имеют роль "admin" в системе
- Доступ к админ-функциям защищен middleware `require_admin`
- Все действия администраторов логируются в AuditLog

### Рабочий процесс администратора

#### Создание нового учебного материала
1. Создать Unit (учебный блок)
2. Добавить Lessons к Unit
3. Создать Exercises для каждого Lesson
4. Опубликовать контент (published = true)

#### Мониторинг активности
1. Просмотреть статистику через `/api/v1/admin/stats`
2. Проверить аудит-логи через `/api/v1/admin/logs`
3. Отследить прогресс конкретных детей

#### Управление пользователями
1. Просмотреть всех пользователей
2. При необходимости деактивировать аккаунты
3. Мониторить подозрительную активность

### Требования к контенту
- Уроки должны иметь четкие названия и описания
- Упражнения должны соответствовать типу урока
- Контент должен быть организован по уровням сложности
- Материалы должны быть проверены на корректность

### Аудит и compliance
- Все изменения контента логируются
- Действия администраторов отслеживаются
- Регулярные проверки безопасности
